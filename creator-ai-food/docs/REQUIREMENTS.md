# 需求文档 - Creator AI Food

## 1. 项目概述

### 1.1 背景
帮助用户创造新菜，突破传统菜谱限制，激发烹饪灵感。

### 1.2 目标用户
- 想尝试新菜但缺乏灵感的用户
- 家里有食材不知道怎么搭配的用户
- 喜欢探索创意料理的美食爱好者

### 1.3 核心价值
- 低成本验证"新菜创造"概念
- 社区驱动的真实评价体系
- 简单易用的交互体验

---

## 2. 功能需求

### 2.1 新菜创造（主功能）

#### 输入
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 主食材 | 多选(1-2个) | 是 | 从 49 种食材中选择 |
| 口味偏好 | 单选 | 否 | 咸鲜/酸甜/麻辣/清淡/融合 |
| 难度等级 | 单选 | 否 | 新手/进阶/挑战 |

#### 输出
| 字段 | 说明 |
|------|------|
| 菜品名称 | AI 生成的创意菜名 |
| 食材形态 | 每种主食材的形态建议 |
| 处理方式 | 每种主食材的处理方法 |
| 辅料/调料 | 推荐搭配 |
| 创意亮点 | 为什么这样组合 |
| 可行性评分 | 1-5 分 |

#### 交互规则
- 动态限制：选择豆腐后自动隐藏牛奶
- 存疑食材提供百科链接

### 2.2 经典菜推荐（辅功能）

#### 流程
1. 用户选择菜系（八大菜系）
2. 展示 3-5 道经典菜
3. 点击查看标准食材清单

#### 八大菜系
- 川菜：麻辣鲜香
- 鲁菜：咸鲜为主
- 粤菜：清淡鲜美
- 苏菜：口味偏甜
- 浙菜：鲜嫩软滑
- 闽菜：鲜香清淡
- 湘菜：香辣浓鲜
- 徽菜：重油重色

### 2.3 社区评分系统

#### 用户上传
- 图片：成品照片
- 评分：1-5 星（单维度）
- 可选：简短评价

#### 动态展示规则
| 平均分 | 展示状态 |
|--------|----------|
| ≥ 4.0 | 精选推荐区 |
| 3.0-4.0 | 正常展示 |
| 2.0-3.0 | 待改进区 |
| < 2.0 | 自动删除 |

## 3. 数据需求

### 3.1 食材库（49 种）

#### 分类
| 类别 | 数量 |
|------|------|
| 蔬菜 | 15 |
| 肉类 | 12 |
| 海鲜 | 10 |
| 豆制品/蛋奶 | 6 |
| 主食 | 6 |

#### 数据结构
```json
{
  "id": 1,
  "name": "豆腐",
  "category": "豆制品",
  "texture": "嫩滑",
  "cooking": ["煎", "炸", "炖", "凉拌"],
  "avoid": ["牛奶"],
  "pair_well": ["肉末", "鱼", "蛋"],
  "baike_url": "https://baike.baidu.com/item/豆腐"
}
```

### 3.2 经典菜库

每道菜包含：
- 菜名
- 所属菜系
- 主料清单
- 辅料清单
- 调料清单
- 简要步骤

---

## 4. 非功能需求

### 4.1 性能
- 页面加载 < 3 秒
- AI 响应 < 5 秒

### 4.2 兼容性
- 支持主流浏览器（Chrome、Firefox、Safari、Edge）
- 移动端适配

### 4.3 安全
- API Key 不暴露到前端
- 用户上传图片需审核

---

## 5. 技术约束

### 5.1 成本控制
- 使用免费/低成本服务
- MVP 阶段控制 API 调用量

### 5.2 可扩展性
- 代码结构便于后续扩展
- 数据结构预留扩展字段
